## Base

## Alcance

- Rama de trabajo para nuevas funciones y correccion de errores posteriores a `V-01.03`.
- Aplicacion Atlas Balance empaquetable para Windows x64.
- Backend ASP.NET Core 8, frontend React/Vite y Watchdog.
- Paquetes de release en `Atlas Balance/Atlas Balance Release`.
- Documentacion centralizada en `Documentacion`.

## Prioridad

- Corregir bugs antes de sumar funciones vistosas.
- Mantener cada cambio trazado en tickets, bugs o documentacion tecnica.
- No publicar release sin build limpio y verificaciones relevantes.
- No meter artefactos de release en commits normales salvo que el objetivo sea publicar paquete.

## Instrucciones para cambios

- Asociar cualquier cambio nuevo a `V-01.04`.
- Actualizar `Documentacion/DOCUMENTACION_CAMBIOS.md` en cada sesion.
- Actualizar `Documentacion/DOCUMENTACION_TECNICA.md` si cambia codigo, estructura, scripts o configuracion.
- Actualizar `Documentacion/DOCUMENTACION_USUARIO.md` si cambia instalacion, uso, configuracion o ubicacion de paquetes.
- Consultar `Documentacion/LOG_ERRORES_INCIDENCIAS.md` antes de resolver errores.
- Mantener `Documentacion/REGISTRO_BUGS.md` al dia.
- No documentar contrasenas, tokens ni datos privados.
